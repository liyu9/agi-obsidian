# Prompt 场景案例

## 💼 产品工作场景

### 场景 1：用户访谈提纲生成

**背景**：准备对 B 端客户进行需求调研

**Prompt**：
```markdown
你是一位资深用户研究员，擅长 B 端产品需求挖掘。

任务：为 CRM 系统升级项目设计用户访谈提纲

要求：
1. 访谈对象：销售总监、销售经理、销售代表三个角色
2. 每个角色 5-8 个核心问题
3. 问题要开放式，避免是非题
4. 包含痛点挖掘、使用场景、期望功能三个维度

输出格式：
## 销售总监
- 痛点挖掘：
  - 问题 1
  - 问题 2
...
```

### 场景 2：竞品分析报告

**背景**：快速生成竞品分析框架

**Prompt**：
```markdown
你是一位竞争情报分析师。

请为"智能客服系统"生成竞品分析框架，包含：

1. 市场格局分析
   - 主要玩家列表
   - 市场份额估算
   - 产品定位对比

2. 功能对比维度（至少 10 个）
   - 核心功能
   - 差异化功能
   - 技术能力

3. 定价策略分析
   - 计费模式
   - 价格区间
   - 免费策略

4. SWOT 分析模板

请用表格形式呈现功能对比部分。
```

### 场景 3：PRD 文档撰写

**背景**：将需求转化为标准 PRD

**Prompt**：
```markdown
你是一位资深产品经理，负责编写 PRD 文档。

项目：电商 APP 购物车功能优化

需求要点：
- 支持批量删除商品
- 支持商品收藏功能
- 支持价格变动提醒
- 支持优惠券自动匹配

请生成完整的 PRD，包含：
1. 文档信息（版本历史）
2. 产品概述（目标、范围）
3. 用户故事（As a... I want... So that...）
4. 功能需求（详细功能描述）
5. 非功能需求（性能、安全）
6. 数据埋点需求
7. 验收标准

格式：Markdown，使用层级结构
```

## 📝 日常办公场景

### 场景 4：会议纪要整理

**背景**：将杂乱的会议记录整理成结构化纪要

**Prompt**：
```markdown
请整理以下会议记录，输出标准会议纪要：

原始记录：
"张三说那个功能下周五前要上线，李四提到测试时间不够，王五说可以先上核心功能，非核心的放下一版本，老板说预算要控制在 50 万以内..."

输出格式：
## 会议基本信息
- 时间：
- 参会人员：
- 主持人：

## 核心决策
1. 决策内容
2. 负责人
3. 截止时间

## 待办事项
- [ ] 任务描述 @负责人 #截止日期

## 风险与问题
- 风险描述，影响程度，应对建议
```

### 场景 5：邮件撰写

**背景**：写一封专业的跨部门协作邮件

**Prompt**：
```markdown
请帮我写一封邮件给技术团队：

背景：
- 需要技术团队支持数据接口开发
- 项目时间紧迫，下周五前需要
- 涉及 3 个接口：用户信息、订单数据、支付状态
- 已和产品团队确认需求

语气要求：
- 专业、礼貌
- 说明紧迫性但不施压
- 表达感谢和配合意愿

包含要素：
- 项目背景
- 具体需求
- 时间节点
- 对接人信息
- 感谢语
```

## 🎯 学习提升场景

### 场景 6：概念解释

**背景**：快速理解陌生概念

**Prompt**：
```markdown
请用费曼学习法解释"Transformer 架构"：

1. 用一句话概括核心思想（高中生能懂）
2. 用生活化类比说明工作原理
3. 列出 3 个关键组件及其作用
4. 举例说明实际应用场景
5. 常见误解澄清

要求：
- 避免数学公式
- 多用比喻和图示描述
- 控制在 800 字以内
```

### 场景 7：代码解释

**背景**：理解复杂代码逻辑

**Prompt**：
```markdown
请解释以下 Python 代码的功能：

```python
def calculate_ltv(customers):
    ltv = 0
    for customer in customers:
        avg_order_value = customer['total_revenue'] / customer['order_count']
        purchase_frequency = customer['order_count'] / customer['customer_age_days'] * 365
        customer_lifespan = 365 / (1 + customer['churn_risk'])
        ltv += avg_order_value * purchase_frequency * customer_lifespan
    return ltv / len(customers)
```

请说明：
1. 函数输入输出
2. 核心计算逻辑
3. 每个变量的含义
4. 业务意义
5. 可能的优化点
```

## 🔧 数据分析场景

### 场景 8：SQL 查询生成

**背景**：快速生成数据分析 SQL

**Prompt**：
```markdown
数据库表结构：
- users(user_id, register_date, user_type)
- orders(order_id, user_id, order_date, amount, status)
- products(product_id, product_name, category, price)

请生成以下分析需求的 SQL 查询：

1. 计算每月 GMV 和订单数
2. 按用户类型统计复购率
3. Top10 畅销商品及其销售额
4. 用户生命周期价值（LTV）分群

要求：
- 使用 CTE 提高可读性
- 添加中文注释
- 考虑性能优化
```

### 场景 9：数据可视化建议

**背景**：选择合适的图表展示数据

**Prompt**：
```markdown
我有以下数据集，请推荐最适合的可视化方案：

数据：
- 时间范围：2024 年 1-12 月
- 指标：销售额、订单量、用户数、客单价
- 维度：全国 30 个省份
- 对比：今年 vs 去年同期

需求：
1. 推荐 5 个最合适的图表类型
2. 说明每个图表展示的核心洞察
3. 给出配色建议
4. 推荐工具（考虑易用性）

请给出具体建议并说明理由。
```

## 🚀 创新创意场景

### 场景 10：头脑风暴

**背景**：生成产品创新点子

**Prompt**：
```markdown
背景：为 B 端 SaaS 产品设计 AI 功能

约束条件：
- 目标用户：中小企业
- 行业：零售电商
- 技术可行：基于现有 LLM 能力
- 商业价值：能提升付费转化

请使用 SCAMPER 方法生成创意：
- S（Substitute）：什么可以被替代？
- C（Combine）：什么可以合并？
- A（Adapt）：可以调整什么？
- M（Modify）：可以修改什么？
- P（Put to other uses）：其他用途？
- E（Eliminate）：可以去掉什么？
- R（Reverse）：可以反转什么？

每个维度至少 3 个创意，评估可行性和价值。
```

## 📊 Prompt 效果对比

| 场景 | 无 Prompt 技巧 | 使用 Prompt 工程 | 提升效果 |

| 用户访谈 | 问题散乱，覆盖不全 | 结构化，多维度 | ⬆️ 80% |
| 竞品分析 | 信息碎片化 | 框架完整，对比清晰 | ⬆️ 60% |
| PRD 撰写 | 遗漏关键要素 | 标准化，完整 | ⬆️ 70% |
| 会议纪要 | 需要大量整理 | 结构化输出 | ⬆️ 90% |
| 代码解释 | 逐行理解耗时 | 整体逻辑清晰 | ⬆️ 75% |
