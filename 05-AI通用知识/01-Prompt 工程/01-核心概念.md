# Prompt 工程核心概念

## 📌 什么是 Prompt 工程

Prompt 工程是通过精心设计和优化输入提示（Prompt），引导大型语言模型（LLM）产生期望输出的技术和方法。

### 核心价值
- 🎯 **提升输出质量** - 更准确、更有用的模型响应
- 💰 **降低成本** - 减少试错次数和 Token 消耗
- ⚡ **快速迭代** - 无需训练即可优化模型表现
- 🔧 **零代码定制** - 通过自然语言调整模型行为

## 🎨 核心技术方法

### 1. Zero-shot Prompting（零样本提示）

直接给模型指令，不提供示例。

**示例**：
```
请将以下中文翻译成英文：
"人工智能正在改变世界"
```

**适用场景**：
- 简单明确的任务
- 模型已具备相关知识
- 快速测试

### 2. Few-shot Prompting（少样本提示）

提供少量示例，让模型学习模式。

**示例**：
```
将产品特性转化为用户价值：

产品特性："支持实时协作编辑"
用户价值："团队成员可以同时编辑文档，无需等待，提升协作效率"

产品特性："自动保存历史版本"
用户价值："随时可以恢复到之前的版本，不用担心误操作丢失内容"

产品特性："智能语法检查"
用户价值：
```

**适用场景**：
- 需要特定格式输出
- 复杂任务模式学习
- 统一输出风格

### 3. Chain of Thought（CoT，思维链）

引导模型展示推理过程。

**示例**：
```
问题：小明有 5 个苹果，他给了小红 2 个，又买了 3 个，现在有几个？

请一步步思考：
1. 小明初始有 5 个苹果
2. 给小红 2 个后：5 - 2 = 3 个
3. 又买了 3 个后：3 + 3 = 6 个
4. 所以小明现在有 6 个苹果

答案：6 个
```

**适用场景**：
- 数学计算问题
- 逻辑推理任务
- 复杂决策分析

### 4. ReAct（Reasoning + Acting）

结合推理和行动，让模型使用工具。

**示例**：
```
思考：用户想了解今天的天气
行动：调用天气 API
观察：北京，晴，25°C
思考：根据天气数据，给出穿衣建议
回答：今天北京晴朗，气温 25°C，建议穿薄长袖或短袖，外出无需带伞。
```

**适用场景**：
- 需要外部数据
- 多步骤任务执行
- Agent 系统设计

## 🛠️ Prompt 设计框架

### CRISPE 框架

| 要素 | 说明 | 示例 |
|------|------|------|
| **C**ontext（背景） | 提供任务背景信息 | "你是一位资深产品经理" |
| **R**ole（角色） | 定义 AI 角色 | "负责 AI 产品规划" |
| **I**nstruction（指令） | 明确任务要求 | "撰写一份 PRD 文档" |
| **S**tyle（风格） | 指定输出风格 | "专业、简洁、结构化" |
| **P**arameters（参数） | 限定输出格式 | "包含用户故事、功能列表、验收标准" |
| **E**xamples（示例） | 提供参考样例 | "参考以下 PRD 模板..." |

### 完整示例

```markdown
# Context
你是一位资深 AI 产品经理，有 5 年 B 端产品设计经验

# Role
负责为公司设计一款智能客服系统

# Instruction
请完成以下任务：
1. 分析目标用户群体
2. 列出核心功能清单
3. 设计产品架构

# Style
- 专业、逻辑清晰
- 使用结构化表达
- 避免技术术语堆砌

# Parameters
- 输出格式：Markdown
- 字数：1500 字左右
- 包含：用户画像、功能矩阵、技术架构图

# Examples
参考以下用户画像格式：
"姓名：张客服，28 岁，客服主管，痛点：..."
```

## 📊 Prompt 优化技巧

### 1. 明确具体
❌ "写一份报告"
✅ "写一份 2024 年 Q4 产品销售分析报告，包含销售额、增长率、Top10 产品"

### 2. 分步引导
❌ "分析这个产品"
✅ "第一步：分析产品定位；第二步：分析目标用户；第三步：分析核心功能"

### 3. 设定约束
❌ "介绍一下 AI"
✅ "用 300 字向高中生介绍 AI，避免使用专业术语，多用生活化比喻"

### 4. 提供角色
❌ "帮我写代码"
✅ "你是一位资深 Python 工程师，擅长编写高性能、可读性强的代码"

### 5. 迭代优化
```
V1: "生成产品需求文档"
V2: "生成电商 APP 的产品需求文档"
V3: "作为资深产品经理，生成电商 APP 的 PRD，包含用户故事、功能列表、原型图描述"
```

## 🔬 高级技巧

### 1. 自我一致性（Self-Consistency）
让模型多次生成，选择最优答案。

```
请回答以下问题，生成 3 个不同版本的答案，然后选择最好的一个：
问题：...
```

### 2. 生成知识（Generated Knowledge）
先生成相关知识，再回答问题。

```
首先，列出回答这个问题需要了解的 5 个关键知识点
然后，基于这些知识回答问题：...
```

### 3. 反思提示（Reflection Prompting）
让模型检查和改进自己的输出。

```
请完成以下任务，然后：
1. 检查你的答案是否存在逻辑错误
2. 评估是否完整回答了问题
3. 如果有问题，请修正并说明原因
```

## 📈 评估指标

| 指标 | 说明 | 如何衡量 |
|------|------|----------|
| **准确性** | 输出是否正确 | 人工评审、自动化测试 |
| **一致性** | 多次输出是否稳定 | 重复测试方差 |
| **完整性** | 是否覆盖所有要点 | 检查清单对照 |
| **简洁性** | 是否冗余 | Token 数量、可读性 |
| **可用性** | 是否可直接使用 | 用户反馈、采用率 |
