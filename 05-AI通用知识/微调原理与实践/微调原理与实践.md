# 微调原理与实践：千层饼加酱

## 企业为什么需要微调？

### 三大原因
1. **安全性**：不能把API schema放网络上"裸奔"
2. **准确性**：专属业务数据微调后准确率提升约10%
3. **成本**：长期看私有化部署成本更低

### 成本分析
- 红线：通用大模型（如GPT4o）成本
- 绿线：私有化部署模型成本
- 私有化起点高（OA段：几千-几万元）
- 但使用量增长到B点后，私有模型成本更优

### 模型选择
- 参数量较小的模型经过微调可能接近通用大模型
- 像狗狗大脑在嗅觉专属领域超过人脑一样

## 微调原理：千层饼加酱

### 千层饼 = 神经网络

**矩阵 = 千层饼的一层**
- M × N个管道宽度值 = 模型参数
- 巨大神经网络 = 多层矩阵组成

### 涂酱 = 改变参数

| 方法 | 说明 |
|------|------|
| 全参微调 | 改变所有参数，像烤制整个饼 |
| 参数高效微调（LoRA）| 只改变少数参数，像在某些层抹酱 |

## LoRA微调原理

### 三步走

**第一步：找到涂酱的地方（Target Module）**

参考Huggingface开源模型（如Mistral-7B）：
- embedding层
- lm_head层
- 中间32个Layer，每层有9层

**第二步：如何涂酱**

- 保持基座模型参数不变
- 训练出"增量矩阵"
- 使用时把增量加到原有参数上

**第三步：制作酱料**

**秩（Rank）的概念：**
- 秩 = 矩阵中可抽取、压缩的特征数量
- LoRA训练两个低秩矩阵：R×M 和 R×N
- R远小于M、N，大幅降低训练参数量

### LoRA核心公式
```
参数量：M×N → R×(M+N)，因为R远小于M、N，所以参数量大幅降低
```

### Rank值选择
- 通常为8、16、32、64
- 对模型效果影响不大，需通过实验确定

## 微调数据准备

### 数据特征
1. **遵循企业内部工作流程** → 提升思考力
2. **会使用企业内部工具** → 提升行动力

### 数据格式
本质是many shots，而非few shots

| 类型 | Hard Prompting | Soft Prompting(微调) |
|------|----------------|---------------------|
| 学习方式 | 上下文短期学习 | 长期练习形成肌肉记忆 |
| 参数变化 | 不变 | 发生改变 |

### 数据准备案例

**思考力数据（FireAct项目）：**
```json
{
  "messages": [
    {"role": "user", "content": "问题..."},
    {"role": "assistant", "content": "Thought:...Action:..."},
    {"role": "user", "content": "Observation:..."}
  ]
}
```
效果：GPT3.5微调后准确率从31.4%提升到39.2%

**行动力数据（OpenAI函数调用）：**
```json
指令：让无人机飞上天空
输出：{'name': 'takeoff_drone', 'arguments': '{"altitude": 100}'}
```
效果：86组数据微调后准确率从60%提升到100%

### 数据准备要点

| 要点 | 说明 |
|------|------|
| 覆盖全面 | 所有路径都要覆盖 |
| 数量充足 | 以千为数量级准备 |
| Negative case | 准备corner case提升鲁棒性 |
| 基座模型 | 有推理要求建议70B参数量起步 |

## 模型评估

### 思考力评估
参考"三看一测"中的一测方法

### 行动力评估（AST方法）
1. 对比function选择是否正确
2. 检查必填字段是否缺失
3. 检查是否有多余字段
4. 检查数据类型是否一致
5. 检查数据值是否一致
