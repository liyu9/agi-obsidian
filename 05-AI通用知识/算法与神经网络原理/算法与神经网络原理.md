# 算法与神经网络原理

## Karpathy的软件2.0理论

### 核心观点
> 神经网络代表了软件开发方式的一次根本性转变，这便是软件2.0。在软件1.0，源代码是由程序员编写……在软件2.0中，源代码就是数据集和神经网络。

### 7年后的验证
> 100%的软件2.0计算机，只有一个神经网络，完全没有任何传统软件。

## 算法与神经网络

### 神经网络原理
把神经网络想象成一个水流网络：
- **蓄水池**：代表神经元
- **管道**：代表神经元之间的信息传输通道
- **管道宽度**：模型参数值

### 激活函数
在杯子的一定高度有个孔，如果上游注入的水（信号）累积到这个孔的位置，神经元就被激活。

这模拟人脑思考机制：同样一句话对两个人的刺激可能完全不一样，因为大脑内部神经元之间的参数不同。

## 神经网络的训练过程

### 前向传播 + 反向传播

1. **前向传播**：从A、B、C、D四个水管注入水，希望大部分水从E、G两个水管输出
2. **反向传播**：将期待值与实际输出值的差异作为信号向上游反馈

> "每次纠正一点进行尝试，然后根据每一次尝试的结果与期待结果的差异不断调整"

### 数据定义产品

**Software 1.0方式：**
```
如果客户标签是"新生儿宝妈"，就推荐"纸尿裤"
如果客户标签是"幼儿园宝妈"，就推荐"书包"
```

**Software 2.0方式：**
构建数据集（一万对"新生儿宝妈→纸尿裤"数据对），一次性扔给神经网络，自动调整参数。

## 数据 vs 编程语言

| 方式 | 说明 |
|------|------|
| 规则定义 | 如果...那么...，导致"智障"客服 |
| 数据定义 | 优秀问答对训练模型，实现"智能客服" |

## GPT的诞生

- **Generative Pretrained Transformer**
- P-Pretrained：预先训练好的意思
- 参数量足够大，训练数据足够多
- 能适配各种产品和场景

## 大语言模型的选择

### 三看一测

| 方法 | 说明 |
|------|------|
| 一看：榜单 | 指定能力测验榜单、Arena群众反馈榜单 |
| 二看：训练数据 | 通过训练数据理解模型能力 |
| 三看：模型厂商应用和客户 | 根据核心应用和客户群判断擅长场景 |
| 一测：实际场景测试 | 准备测试数据集，批量运行评估 |

### 模型榜单类型

| 类型 | 说明 |
|------|------|
| 指定能力测验榜单 | 统一出题，分门别类测试（SuperCLUE理科卷、文科卷、Hard卷） |
| Arena群众反馈榜单 | 随机两个模型PK，群众投票 |

### 测试数据集准备
- 确定业务测试数据集：多组输入输出数据对
- 输入是提示词，输出是期待大模型输出的结果

### 评测方法
- **机器评测**：大语言模型当裁判打分
- **用户反馈**：灰度放量期间按用户满意度评判
