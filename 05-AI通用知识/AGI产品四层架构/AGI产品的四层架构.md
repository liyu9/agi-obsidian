# AGI产品的四层架构

## 核心框架

AI产品如同冰山，用户感受到的只是浮出水面的十分之一，隐藏在水下的十分之九才是AI产品经理的重头戏。

**AI产品四层结构：**
| 层级 | 名称 | 类比 |
|------|------|------|
| 第一层 | 数据、算力、算法层 | 教育体系（学习资源、教学设备、老师） |
| 第二层 | 模型层 | 培养出的学生 |
| 第三层 | 应用层 | 元器件的组织协调 |
| 第四层 | 用户层 | 产品的外在形象 |

## 第一层：数据、算力、算法层

### 数据
数据分为两类：
- **基础数据**：用于建设AI基础能力的海量数据，由大模型厂商决定
- **个性化数据**：AI产品经理需要"精挑细选"的数据

AI产品经理需了解三个问题：
1. 判断什么样的数据是优质数据？
2. 如何获取这些优质数据？
3. 这些数据对AI产品会产生哪些影响？

### 算力
支撑AI产品的硬件环境（计算芯片）。

产品经理需了解：
1. 如何评估一个AI产品对算力的需求？
2. 使用这些算力要花多少钱？
3. 使用哪些方法可以降低成本？

### 算法
AI背后最抽象的计算机原理。

了解算法的两大好处：
1. 理解技术边界，鉴定什么问题AI能解决/不能解决
2. 与算法工程师、开发工程师更好地合作

## 第二层：模型层

模型层可分为三层，对应不同水平的学生：

| Level | 类型 | 说明 |
|-------|------|------|
| Level 0 | 基座大模型 | 本科生，具备通用能力 |
| Level 1 | 垂直领域模型 | 研究生，具备专业领域知识 |
| Level 2 | 场景模型 | 业务专家，具备某些业务专家能力 |

AI产品经理如同公司HR，需具备：
1. 如何挑选合适的员工——为产品场景找到合适的基座大模型或领域模型
2. 如何进行公司内部的专项培训——训练场景模型，理解模型微调原理

## 第三层：应用层

应用层是产品经理工作最繁重的地方，需掌握两大核心技术：

### 提示词工程

AI产品经理写提示词 vs 普通用户写提示词：
- **普通用户**：像玩盲盒，对结果有所期待但不要求确定
- **AI产品经理**：像打靶，有确定目标，直到提示词能打中靶心

提示词工程两种类型：
- **Hard prompting（硬提示）**：把提示词当做输入项直接给到模型
- **Soft prompting（软提示/Prompt tuning）**：用微调模型的方法让模型学习某种模式

### Agent设计

Agent（智能体）在思考基础之上增加行动能力。

**Agent五个等级：**
| 等级 | 能力 |
|------|------|
| 第一级 | 基于规则，能使用工具的智能体 |
| 第二级 | 基于模仿学习、强化学习，能使用工具且能做决策、推理 |
| 第三级 | 基于大语言模型，增加记忆、反思能力 |
| 第四级 | 基于大语言模型，增加自主学习能力 |
| 第五级 | 基于大语言模型，增加情感、社会协同能力 |

**Agent四项核心能力：**
| 能力 | 说明 |
|------|------|
| 工具使用能力 | 调用外部工具完成任务 |
| 记忆能力 | 记住对话上下文 |
| 反思能力 | 以特定角色评判结果 |
| 规划决策能力 | 自主规划解决路径 |

## 第四层：用户层

人类的信息有80%是从视觉中获得的。

**AI产品UX两个原则：**
1. 让用户的学习路径最短
2. 让效果呈现最直观

## 四层架构总结

| 层级 | 决定因素 | 产品经理职责 |
|------|----------|--------------|
| 最底层 | 生命体的气质 | 确定技术边界，决定什么功能可用AI实现 |
| 模型层 | 生命体的专业素养 | 选用合适基础模型，收集优质数据集 |
| 应用层 | 培训生命体的实战技术 | 掌握提示词工程和Agent设计 |
| 用户层 | 塑造生命体的外在形象 | 缩短用户学习路径，增强效果呈现 |
