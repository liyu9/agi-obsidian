# Agent设计模式

## Agent四大核心能力

Agent如同大语言模型的外围设备，支撑更可靠的发挥：

| 能力 | 作用 | 实现方式 |
|------|------|----------|
| 记忆能力 | 记住上下文和规则 | Hard Prompting、Soft Prompting、RAG |
| 工具使用能力 | 执行具体操作 | Re-Act模式 |
| 规划能力 | 制定解决步骤 | Plan-and-Solve模式 |
| 反思能力 | 检查核对结果 | Reflection模式 |

## 记忆能力四象限

|  | 事实性记忆 | 程序性记忆 |
|--|------------|------------|
| **短期记忆** | 上下文信息 | 对话风格 |
| **长期记忆** | 客观事实 | 业务规则 |

## Re-Act模式

**Reasoning and Act**：推理并且执行

适用场景：外界环境不可预见性强，任务比较单一

提示词模板：
```
## Instruction
用思考、行动、观察交错的步骤解决问答任务。
- Search[entity]：在Wikipedia上搜索精确实体
- Lookup[keyword]：返回当前段落中包含关键字的下一个句子
- Finish[answer]：返回答案并完成任务
```

## Plan-and-Solve模式

受COT启发，列出任务每一步，约等于制定计划。

提示词模板：
```
给定一个任务，我们首先了解该任务并制定完成任务的计划。
然后，让我们执行该计划，您可以使用以下工具执行步骤：
- 搜索[您的关键字]
- 总结[内容]
- 完成[答案]
```

## Reflection模式

如同老师检查作业，给Checker的提示词需包含：总结内容、搜索结果、任务说明

提示词模板：
```
给定一个任务，用户已经完成了该任务，
请你根据任务描述，任务完成情况，任务完成依据对任务进行检查，
并提供详细的建议，包括事实检查、核对数据正确性。
```

## Agent设计模式选择指南

| 场景 | 推荐模式 |
|------|----------|
| 知识问答、词条搜索、政策咨询（步骤多、不确定高） | Plan-and-Solve + 反思 |
| 内部生产数据问答（步骤不多、不确定高） | Re-Act |
| 文本数据提取、文档合规审查（任务简单、环境确定） | 反思模式 |
| 公共信息搜索、资讯总结（确定性高、难度低） | 直接使用 |
