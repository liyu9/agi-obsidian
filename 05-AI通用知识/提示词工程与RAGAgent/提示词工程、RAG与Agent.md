# 提示词工程、RAG与Agent

## 一、提示词工程

### AI产品经理的提示词 vs 普通用户的提示词

| 维度 | 普通用户 | AI产品经理 |
|------|----------|------------|
| 目标 | 玩盲盒，对结果有所期待但不要求确定 | 打靶，有确定目标，直到打中靶心 |
| 工具 | ChatGPT等对话产品 | GPT Playground等原生模型调试 |

### 提示词框架：RTF

**RTF = Role（角色）+ Task（任务）+ Format（格式）**

示例：
```
## Role
情感标注人员

## Task
我给你的一句客户留言文字，请你判断出文字中的情绪。请你从以下选项中选择：
1. 正面：表达了肯定、喜欢、赞赏、褒扬
2. 负面：否定、讨厌、贬低
3. 中性：客观描述，无主观意向

## Format
严格按照以下格式输出：
{
  "客户留言": "客户留言内容",
  "情绪判断": "情绪判断结果"
}
```

### 提示词两个核心技巧

#### 技巧一：Few-Shots（少量样本提示）

通过举例让模型"照猫画虎"地学习。

研究发现：
- 给1个示例比不给示例准确率从10%提升到45%
- 给10个示例比1个示例准确率从45%提升到60%
- 超过10个示例效果可能波动下降

**核心：少量样本数量不宜过少，但也不能过多**

#### 技巧二：COT（思维链 Chain Of Thought）

在提示词里加上"让我们一步步来，第一步……，第二步……"

主要用于推理场景：数学计算、时间规划等。

效果：在参数量十亿、百亿、千亿模型上，准确率从20%直接提升到50%

## 二、RAG（检索增强生成）

### RAG的核心作用

把检索（资料库）的结果发给大模型，增强大模型的有据输出，降低"臆想答案"概率。

案例：通过RAG技术将回答准确率从45%提升到98%

### RAG Pipeline流水线

| 环节 | 说明 |
|------|------|
| 1. 数据提取 | 确保资料数据完整，可生成summary、标签 |
| 2. 分块(Chunking) | 按段落、句子等拆分，对齐颗粒度 |
| 3. 向量化(Embedding) | 转换为计算机可处理的向量 |
| 4. 创建索引 | 分类整理，方便检索查找 |
| 5. 检索(Retriever) | 根据用户提问查找相关数据块 |
| 6. 检索评估 | 核心指标：召回率 |
| 7. 重排序(Rerank) | 对初步检索结果进行精排 |
| 8. 重排评估 | 评估重排效果 |
| 9. 生成(Generator) | 将结果发给LLM生成最终输出 |
| 10. 终极评估 | RAGAS框架评估 |

## 三、Agent

### Agent的组成

Agent本身包含了RAG和提示词的应用，核心是让AI具备**思考力+执行力**。

**思考力**：通过提示词工程和RAG获得
**执行力**：通过Function calling和Re-Act获得

### Function calling（函数调用）

把自然语言指令转换为可执行的函数调用指令。

示例：
```
用户请求：查一下今天北京的天气
↓ 大语言模型转换
{
    "function": "get_current_weather",
    "parameters": {"location": "北京"}
}
```

### Re-Act模式

**Reasoning and Act**：推理并且执行，给行动装上随机应变的机制。

提示词模板：
```
## Observation：
API返回错误信息"location must be English"

## Think：
应该将location字段的值转换为英文再发一次

## Act：
{
    "function": "get_current_weather",
    "parameters": {"location": "BeiJing"}
}
```

循环直到达成目标或达到循环上限。
