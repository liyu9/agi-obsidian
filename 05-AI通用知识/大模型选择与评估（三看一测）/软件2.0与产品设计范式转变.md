# 软件2.0与产品设计范式转变

## Karpathy的软件2.0理论

2017年，OpenAI联合创始人Karpathy提出：

> 神经网络代表了软件开发方式的一次根本性转变，这便是软件2.0。在软件1.0，源代码是由程序员编写……在软件2.0中，源代码就是数据集和神经网络。

7年后，Karpathy再写道：

> 100%的软件2.0计算机，只有一个神经网络，完全没有任何传统软件。

## 范式转变

### 软件1.0 vs 软件2.0

| 维度 | 软件1.0 | 软件2.0 |
|------|---------|---------|
| 核心要素 | 编程语言、规则 | 数据集、神经网络 |
| 定义方式 | 程序员编写规则 | 定义数据对 |
| 代表 | 传统软件 | 神经网络应用 |

### 神经网络训练原理

**前向传播 + 反向传播：**
1. 给出一组成对的输入、输出值
2. 神经网络进行"前向传播"，输出结果
3. 结果与期待值的差异"反向传播"反馈
4. 重复调整参数，直到输出符合期待

### 数据定义产品举例

**软件1.0方式：**
```
如果客户标签是"新生儿宝妈"，就推荐"纸尿裤"
如果客户标签是"幼儿园宝妈"，就推荐"书包"
```

**软件2.0方式：**
构建数据集（可能有一万对"新生儿宝妈→纸尿裤"的数据对），一次性扔给神经网络自动调整参数。

## 大语言模型的选择

### 选择模型三要素

| 要素 | 说明 |
|------|------|
| 是否合规可用 | 参考国家生成式人工智能备案信息 |
| 模型成本 | 闭源API按token消耗，开源看私有化部署成本 |
| 能力匹配度 | 通用能力+专项能力是否符合应用场景 |

### 三看一测法

#### 一看：榜单

| 榜单类型 | 说明 |
|----------|------|
| 指定能力测验榜单 | 统一出题，分门别类测试（SuperCLUE理科卷、文科卷、Hard卷） |
| Arena群众反馈榜单 | 随机两个模型PK，群众投票 |

#### 二看：训练数据

通过训练数据理解模型能力（目前大部分主流模型训练数据未开源）

#### 三看：模型厂商核心应用和客户

根据模型厂商的象征性应用和核心客户群判断模型擅长场景

#### 一测：实际场景测试

1. **准备测试数据集**：多组输入输出数据对
2. **批量运行**：通过工程化方式测试
3. **评估方法**：
   - 机器评测：大语言模型当裁判打分
   - 用户反馈：在灰度放量期间按用户满意度评判
