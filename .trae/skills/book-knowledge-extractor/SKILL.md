# 书籍知识提取器

## 功能概述

将书籍PDF内容系统性地提取、分类、入库到知识框架。

---

## 执行流程

### Step 1: PDF 检测

**检查PDF类型：**
```python
import fitz
doc = fitz.open(pdf_path)
page = doc[0]
images = page.get_images()
# 如果每页都有图片，说明是扫描件，需要OCR
```

**处理方案：**
| PDF类型 | 处理方式 |
|---------|----------|
| 文字层可用 | 直接用 pdfplumber 提取 |
| 扫描件 | 用 EasyOCR 逐页OCR |

### Step 2: 目录结构分析

**优先提取内容：**
- 前20页通常包含目录、序言、作者简介
- 识别章节结构和主要话题

**提取命令：**
```python
reader = easyocr.Reader(['ch_sim', 'en'], gpu=False)
for i in range(20):
    page = doc[i]
    pix = page.get_pixmap(dpi=200)
    results = reader.readtext(img_path)
    text = '\n'.join([t for _, t, _ in results])
```

### Step 3: 分批次处理

**批次规则：**
- 每50页为一批
- 每批处理前先输出"更新计划"
- 用户确认后再执行

**OCR优化：**
- DPI选择：200（平衡速度与质量）
- 语言：中文（chi_sim）+ 英文（en）
- 图片临时保存后立即删除

### Step 4: 知识分类

**分类原则：优先匹配现有目录**

| 内容类型 | 归类目录 | 示例 |
|----------|----------|------|
| 用户研究、需求调研、业务调研 | `01-产品通用知识/01-用户研究与需求调研/` | 业务调研六要点 |
| 产品设计理念 | `01-产品通用知识/02-产品设计/` | 设计工程化 |
| PRD 撰写、方案编写 | `01-产品通用知识/03-PRD 文档撰写/` | 方案编写方法 |
| 产品规划、从 0 到 1 | `01-产品通用知识/03-产品规划与文档体系/` | 产品规划方法 |
| 竞品分析、行业分析 | `01-产品通用知识/04-行业分析与竞品分析/` | 竞品分析三维度 |
| 业务认知、行业知识 | `02-业务知识/` | SaaS 基础概念 |
| 职业素养、软技能 | `09-软技能与职业成长/` | 产品经理 6 大素养 |
| 思维模型、方法论 | `03-思维能力与方法论/` | 框架效应 |
| 管理知识 | `04-管理知识/` | 团队管理 |
| AI/新技术 | `05-AI 通用知识/` 或 `07-AI 新技术与工具/` | AI 实践 |

**目录命名规范：**
- 优先使用现有目录（带序号）
- 如现有目录不匹配，再创建新目录（格式：`分类名（关键词 1/关键词 2/...）`）

**文件命名规范：**
- 不加数字序号，便于扩展
- 格式：`主题名.md`
- 示例：`业务调研六要点.md`、`竞品分析三维度.md`

### Step 5: 内容提取要求

**核心要素提取：**
- 章节标题和小标题
- 关键概念定义
- 核心观点（1、2、3...）
- 重要案例（简化描述）
- 方法论/框架/模型
- **图片/图表/流程图**（必须保存）

**图片处理规范：**
1. **保存位置**：对应目录的 `assets/` 子目录
   - 示例：`01-产品通用知识/04-行业分析与竞品分析/assets/`
2. **命名格式**：`YYYYMMDD-序号 - 描述.png`
   - 示例：`20260330-01-竞品分析三层模型.png`
3. **正文引用**：使用相对路径
   - 示例：`![竞品分析三层模型](assets/20260330-01-竞品分析三层模型.png)`
4. **提取方式**：
   ```python
   # 提取页面中的图片
   img_list = page.get_images()
   for img_info in img_list:
       xref = img_info[0]
       base_image = doc.extract_image(xref)
       image_bytes = base_image["image"]
       # 保存到 assets/目录
   ```

**保持结构化：**
- 使用Markdown标题层级
- 使用表格对比
- 使用列表归纳要点
- 保留关键引用

```markdown
- 摘要：[本章/批次的核心内容]

### 操作清单
| 操作 | 目标路径 | 内容概要 |
|------|----------|----------|
| 新建 | XX/分类名（）/ | 分类说明 |
| 新增 | XX/分类名（）/文件名.md | 核心要点1/2/3 |

### 风险提示
- ⚠️ [如有新建目录]
- ✅ [无删除/移动操作]
```

## 防护规则

1. **OCR准确性**：关键概念需核对，避免文字识别错误
2. **内容保真**：提取核心观点，不做过度概括
3. **目录变更**：新建目录先输出计划
4. **记录追溯**：每批次完成后更新 `.trae/documents/知识框架更新记录.md`
