# AGI知识框架维护

维护AGI产品经理知识框架的完整性和一致性。

## 知识框架结构

参考 `references/knowledge_domains.md` 了解完整的9大知识域结构：

```
01-产品通用知识/      # 产品经理基础能力
02-业务知识/          # 垂直业务领域理解
03-思维能力与方法论/  # 思维框架和方法论
04-管理知识/          # 团队协作和管理
05-AI通用知识/        # AI技术原理和应用
06-AI落地实践/        # AI产品实战案例
07-AI新技术与工具/    # 前沿AI工具和技术
08-商业化与增长/      # 商业模式和增长策略
09-软技能与职业成长/  # 职业发展和软技能
11-知识管理/          # 知识管理方案
12-学习笔记/          # 学习笔记和课程
```

## 工作流程

### Step 1: 接收与解析信息

接收用户输入的信息：
- 文本内容
- URL链接
- 文件路径
- 知识内容描述

**解析要点**：
- 识别核心主题
- 提取关键知识点
- 确定内容类型
- 评估内容价值

### Step 2: 映射知识域

将内容对标到9个一级知识域：

**映射规则**：
- 用户研究 → `01-产品通用知识/01-用户研究与需求调研/`
- 产品设计 → `01-产品通用知识/02-产品设计/`
- PRD撰写 → `01-产品通用知识/03-PRD文档撰写/`
- 业务知识 → `02-业务知识/`
- 思维方法 → `03-思维能力与方法论/`
- 管理知识 → `04-管理知识/`
- AI技术 → `05-AI通用知识/`
- AI实践 → `06-AI落地实践/`
- AI工具 → `07-AI新技术与工具/`
- 商业增长 → `08-商业化与增长/`
- 软技能 → `09-软技能与职业成长/`

**映射原则**：
1. **主题匹配**：根据内容主题确定一级知识域
2. **层级细分**：在一级域下选择合适的二级子目录
3. **灵活扩展**：必要时创建新的二级或三级目录

### Step 3: 生成更新计划

参考 `references/protection_rules.md` 防护规则，生成详细更新计划：

```markdown
## 更新计划

### 输入信息
- **来源**：[用户输入/URL/文件路径]
- **摘要**：[内容摘要]
- **主题**：[核心主题]
- **类型**：[文章/笔记/案例/方法论]

### 知识域映射
- **一级知识域**：[知识域名称]
- **二级目录**：[子目录名称]
- **目标路径**：[完整路径]

### 操作清单

| 操作 | 类型 | 目标路径 | 文件名 | 原因 |
|------|------|----------|--------|------|
| 新增 | 文件 | XX/YY/ | 内容.md | 补充XX知识 |
| 新建 | 目录 | XX/YY/新目录/ | - | 分类需要 |
| 移动 | 文件 | XX/YY/ | 旧文件.md | 调整归类 |
| 保存 | 图片 | XX/assets/YY/ | 图片.png | 图片资源 |

### 图片资源处理
- **图片数量**：[数量]
- **存放位置**：`[知识域]/assets/[主题]/`
- **命名规范**：`[主题]_[类型]_[序号].png`

### 文本保真度保护
- **原文保留**：核心内容完整保留
- **格式转换**：转换为Markdown格式
- **结构调整**：按知识框架重组结构

### 目录变更说明
- **是否需要新建目录**：[是/否]
- **新建目录路径**：[路径]
- **新建原因**：[原因]

### 预期影响
- **新增文件数**：[数量]
- **影响现有文件**：[文件列表]
- **需要更新链接**：[链接列表]
```

### Step 4: 执行更新

**⚠️ 重要**：必须等待用户确认后再执行！

#### 文件操作

**新增文件**：
1. 创建Markdown文件
2. 应用内容模板
3. 添加来源标注
4. 保存到目标路径

**修改文件**：
1. 读取现有文件
2. 保留原有内容
3. 添加新内容
4. 更新来源标注

**移动文件**：
1. 确认目标路径
2. 移动文件到新位置
3. 更新内部链接
4. 记录移动原因

**删除文件**：
1. 确认删除必要性
2. 备份文件内容
3. 删除文件
4. 更新相关链接

#### 目录操作

**新建目录**：
1. 创建目录结构
2. 创建assets子目录（如需要）
3. 添加目录说明文件

**删除目录**：
1. 确认目录为空
2. 移动或删除文件
3. 删除空目录

#### 图片处理

**保存图片**：
- 存放位置：对应模块的 `assets/` 子目录
- 命名规范：`[主题]_[类型]_[序号].png`
- 路径引用：相对路径引用

### Step 5: 记录变更

```markdown

- **变更原因**：[原因]

### 详细变更

#### 文件变更

   - **变更类型**：新增
   - **变更原因**：补充XX知识
   - **内容摘要**：[摘要]
   - **来源**：[来源信息]

2. **文件路径**：`XX/YY/旧文件.md`
   - **变更类型**：移动
   - **原路径**：`AA/BB/旧文件.md`
   - **新路径**：`XX/YY/旧文件.md`
   - **变更原因**：调整归类

#### 目录变更
1. **目录路径**：`XX/YY/新目录/`
   - **变更类型**：新建
   - **变更原因**：分类需要
   - **子目录**：assets/

#### 图片变更
1. **图片路径**：`XX/assets/YY/图片.png`
   - **变更类型**：新增
   - **图片来源**：[来源]
   - **用途说明**：[说明]

### 影响范围
- **受影响文件**：[文件列表]
- **需要更新链接**：[链接数量]

### 执行状态
- ✅ 计划已确认
```

## 防护规则执行

### 1. 图片资源防护

**规则**：图片必须存放在对应模块的 `assets/` 子目录

**执行检查**：
- [ ] 图片路径正确：`[知识域]/assets/[主题]/`
- [ ] 图片命名规范：`[主题]_[类型]_[序号].png`
- [ ] 图片引用正确：使用相对路径
- [ ] 图片文件存在：确认文件可访问

### 2. 文本保真度防护

**规则**：不得删减、过度概括原文内容

**执行检查**：
- [ ] 核心概念完整保留
- [ ] 原文观点未被改变
- [ ] 重要信息未被删减
- [ ] 格式转换正确合理

### 3. 目录变更控制

**规则**：任何目录增删必须先输出计划

**执行流程**：
1. 输出目录变更计划
2. 说明变更原因和影响
3. 等待用户确认
4. 确认后执行变更

### 4. 文件变更控制

**规则**：必须记录文件变更原因

**执行检查**：
- [ ] 变更原因已说明
- [ ] 变更影响已评估
- [ ] 变更日志已记录
- [ ] 相关链接已更新

## 内容模板

### 新增文件模板

```markdown
---
tags:
  - [知识域]
  - [主题]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---

# [标题]

## [核心内容]

[内容正文]

---
**来源**：[来源信息]
**入库日期**：YYYY-MM-DD
**维护工具**：agi-knowledge-maintainer
---
```

### 来源标注规范

每个文件底部必须包含：

```markdown
---
**来源**：[来源类型] - [具体来源]
**入库日期**：YYYY-MM-DD
**维护工具**：agi-knowledge-maintainer
---
```

**来源类型**：
- 书籍：《书名》第X章
- 文章：[文章标题] - [网站/平台]
- 课程：[课程名] - 第X节
- 实践：[项目/案例名称]
- 总结：个人总结/思考

```markdown
### 输入信息
```

## 触发条件

当用户发送以下内容时自动触发：
- 产品/AI相关文章、笔记、文档
- URL或文件路径
- 要求入库的知识内容
- 请求更新知识框架的操作
- "入库这个内容"、"归类这个文件"等类似表述

## 注意事项

1. **先计划后执行**：任何变更必须先输出计划
2. **用户确认优先**：等待用户确认再执行
3. **防护规则严格执行**：确保图片、文本、目录、文件变更符合防护规则
4. **完整记录变更**：每次变更都要详细记录
5. **及时更新关联**：变更后更新相关链接和索引
