# RAG技术与知识库机器人

## RAG技术全景

### RAG定义
Retrieval Augmented Generation，检索增强生成

### 核心价值
把检索结果发给大模型，增强有据输出，降低"臆想答案"概率

### 效果对比
案例：通过RAG技术将回答准确率从45%提升到98%

## RAG Pipeline流水线

| 环节 | 说明 | 关键点 |
|------|------|--------|
| 1. 数据提取 | 确保资料数据完整 | 可生成summary、标签 |
| 2. 分块(Chunking) | 按段落、句子等拆分 | 对齐颗粒度 |
| 3. 向量化(Embedding) | 转换为向量 | 选择合适模型 |
| 4. 创建索引 | 分类整理 | 方便检索查找 |
| 5. 检索(Retriever) | 根据用户提问查找 | 核心：召回率 |
| 6. 检索评估 | 评估检索效果 | 召回率指标 |
| 7. 重排序(Rerank) | 精排初步结果 | 提升准确率 |
| 8. 重排评估 | 评估重排效果 | - |
| 9. 生成(Generator) | LLM生成输出 | 提示词设计 |
| 10. 终极评估 | RAGAS评估 | 端到端效果 |

## 知识库机器人三大病症

### 病症一：明明有答案却回答不出来

**原因1：检索方式不合理**
| 检索方式 | 特点 |
|----------|------|
| 向量检索 | 同义词距离近，可能无法匹配完整句子 |
| 关键字检索 | 完全匹配，比较死板 |
| 混合检索 | 综合两者，推荐使用 |

**原因2：分块不合理**
- 返回的文本块只有问题，没有回答

**解法：**
| 方法 | 说明 | 代价 |
|------|------|------|
| 冗余重合内容 | 下一个块包含上一个的15% | 增加存储 |
| 强制返回相邻块 | 返回块2时同时返回1和3 | Token成本×2-3 |

### 病症二：如何把图片带进回答

| 方法 | 说明 |
|------|------|
| 图文关联法 | 文本分块设置Metadata包含ImageURL |
| 图文转换法 | 多模态大模型将图片转为文本 |

### 病症三：Token消耗太快

| 方法 | 效果 |
|------|------|
| 问答对匹配代替RAG | 节省80%成本 |
| 提示词压缩 | 压缩30%，可能影响准确率 |

## 最佳实践

### 检索优化
1. **混合检索**：向量+关键字
2. **重排序**：Rerank模型提升准确率
3. **查询扩展**：同义词扩展、语义扩展

### 分块策略
1. **固定大小分块**：按Token数切分
2. **语义分块**：按完整语义段落切分
3. **递归分块**：多粒度分块

### 效果评估

| 指标 | 说明 |
|------|------|
| 召回率 | 相关文档被召回的比例 |
| 准确率 | 召回结果中相关的比例 |
| RAGAS | 综合评估生成质量 |
