# 企业级AI搜索与Query处理

## 企业搜索的特点

### vs 普通搜索

| 维度 | 普通搜索 | 企业搜索 |
|------|----------|----------|
| 数据源 | 公开网页 | 内部文档、数据库 |
| 准确性要求 | 较低 | 高（影响业务决策） |
| 召回率 | 优先 | 必须保证 |
| 冷启动问题 | 无 | 有（新系统数据不足） |

## 意图识别

### 什么是意图识别
理解用户查询背后的真实目的

### 意图分类

| 意图类型 | 示例 |
|----------|------|
| 导航意图 | 找某个具体页面 |
| 信息意图 | 了解某个知识 |
| 事务意图 | 完成某个操作 |

### 企业搜索中的意图

| 意图 | 示例问题 |
|------|----------|
| 知识问答 | "如何申请年假？" |
| 文档查找 | "找2023年Q3销售报告" |
| 操作执行 | "帮我创建工单" |

## Query重写

### 为什么需要Query重写

**Query问题：**
| 问题 | 示例 |
|------|------|
| 表达模糊 | "那个报告"→具体报告名 |
| 表述错误 | "张三的Q2报告"→实际是李四的 |
| 口语化 | "我想查下..."→正式表述 |

### 重写技术

| 技术 | 说明 |
|------|------|
| 同义词替换 | "电脑"→"计算机" |
| 扩展补充 | "报销"→"费用报销流程" |
| 纠错 | 自动纠正拼写错误 |
| 标准化 | 统一日期、金额格式 |

## 路由分发

### 什么是路由
根据Query类型分发到不同处理模块

### 路由决策

```mermaid
用户Query → 意图识别 → 不同路由

路由1: 知识问答 → RAG知识库
路由2: 文档查找 → 文档检索
路由3: 操作执行 → API调用
路由4:闲聊 → 大模型对话
```

### 路由策略

| 策略 | 说明 |
|------|------|
| 精确匹配 | 命中关键词直接路由 |
| 模糊匹配 | 计算相似度后路由 |
| 多路召回 | 同时调用多个模块，结果融合 |

## 企业搜索核心架构

### 整体流程

```
用户Query
    ↓
Query预处理（纠错、标准化）
    ↓
意图识别（分类模型）
    ↓
Query重写（扩展、同义词）
    ↓
路由分发（多路召回）
    ↓
知识库/文档库/API
    ↓
结果重排序
    ↓
结果呈现
```

### 关键指标

| 指标 | 说明 |
|------|------|
| 召回率 | 相关结果被召回的比例 |
| 准确率 | 召回结果中相关的比例 |
| MRR | 平均排名倒数 |
| NDCG | 归一化折损累计增益 |
