# AI产品经理简历撰写与面试指南

## 简历两个基本原则

### 原则一：内容真实具体，方法适度拔高

**真实具体：** 通过具体实践陈述事实
**方法拔高：** 体现实践背后的指导思想

示例：
- 真实具体："提示词中采用反思技巧将回答准确率从xx%提升至xx%"
- 方法拔高："理解Agent设计方法，包括反思、规划、记忆、行动能力构建"

关系：前者依赖后者的指导方法得出，既要有"鱼"（具体成果），也要有"渔"（方法论）

### 原则二：遵循常规结构，突出个性强项

**三个3目标：**
- 3秒内找到重点
- 30秒内了解核心优势
- 3分钟内确认是否适合职位

## 简历结构

| 部分 | 占比 | 说明 |
|------|------|------|
| 个人基本信息 | - | 姓名、性别、年龄、求职意向、联系方式 |
| 个人技能 | 重点 | 3秒+30秒吸引HR |
| 工作经历 | 重点 | 3分钟决定面试 |
| 证书学历 | - | 除非特别牛，否则放最后 |

## 个人技能撰写

### 一句话介绍（3秒）

**结构：**<身份定位>，<核心项目成果>，<结果数据>

**示例：**
- "5年B端产品经理经验，负责xxx产品的AI化改造，增值功能在xx天实现xx元营收"
- "2023年毕业的AI原著民，通过Dify搭建xxx智能体，一个月内用户数增长至xx"

### 硬核技能（30秒）

**三个方向：**

| 方向 | 具体技能 |
|------|----------|
| 使用 | 重度使用ChatGPT、Claude；竞品调研与评测 |
| 动手做 | 操作Dify/Coze；提示词工程；RAG调优；Agent设计 |
| 技术理解 | 神经网络原理；LoRA微调原理；模型能力边界 |

### 软实力（可选）

| 能力 | 表述方式 |
|------|----------|
| 快速学习 | x个月内学完xxx，并动手实践xxx |
| 自我驱动 | 主动提出xxx解决方案，在团队内得到认可并推动落地 |

## STAR法则写工作经历

**STAR = Situation + Task + Action + Result**

| 要素 | 说明 |
|------|------|
| Situation | 为什么要有这个产品，希望达成什么 |
| Task | 需要完成什么任务 |
| Action | 做了哪些事情 |
| Result | 量化结果 |

**量化指标优先级：**
1. 商业指标：订单量、订单金额
2. 用户指标：总用户数、新增、日活、月活
3. 参与度：停留时长、对话次数等

## 面试题类型

### 技术理解类
- 神经网络原理是什么？
- Transformer架构了解吗？
- 大语言模型的能力边界在哪里？

### 产品设计类
- 你怎么发现这个需求是真需求？
- 产品设计中遇到的最大挑战是什么？
- 如何评估AI产品的效果？

### 实战经验类
- 做过哪些AI产品的项目？
- 提示词调优的过程是怎样的？
- 如何处理AI产品的边界case？
