# UML建模与需求建模

## 6.3 通过UML进行需求建模

### UML定义
- **UML**：Unified Modeling Language，统一建模语言
- 用于软件设计和需求分析的可视化建模语言

### UML图分类

| 类型 | 图 | 用途 |
|------|-----|------|
| 结构图 | 类图 | 描述类和数据模型 |
| 结构图 | 对象图 | 描述对象实例 |
| 结构图 | 包图 | 描述模块组织 |
| 行为图 | 用例图 | 描述系统功能需求 |
| 行为图 | 时序图 | 描述对象交互顺序 |
| 行为图 | 状态机图 | 描述对象状态变化 |
| 行为图 | 活动图 | 描述业务流程 |

## 6.3.1 用例图

### 用例图要素

| 要素 | 说明 |
|------|------|
| 参与者(Actor) | 与系统交互的角色 |
| 用例(Use Case) | 系统提供的功能 |
| 关系 | 包含、扩展、泛化 |
| 系统边界 | 系统的作用范围 |

### 用例关系

| 关系 | 说明 | 符号 |
|------|------|------|
| 包含(include) | 用例A包含用例B | <<include>> |
| 扩展(extend) | 用例A扩展用例B | <<extend>> |
| 泛化 | 用例A是用例B的特例 | 箭头 |

### 用例图示例
- 参与者：采购员、运营人员、管理员
- 用例：下单、审核订单、查询库存、客户管理

## 6.3.2 类图与ER图

### 类图要素

| 要素 | 说明 |
|------|------|
| 类名 | 数据对象名称 |
| 属性 | 对象的特征 |
| 方法 | 对象的操作 |

### 类图关系

| 关系 | 说明 | 符号 |
|------|------|------|
| 关联 | 类之间的引用 | 线 |
| 聚合 | 整体与部分 | 空心菱形 |
| 组合 | 强整体与部分 | 实心菱形 |
| 依赖 | 类A依赖类B | 虚线箭头 |
| 继承 | 类A继承类B | 实线三角箭头 |

### ER图（实体关系图）
- 用于描述业务数据模型
- 实体之间的关系

### ER图关系表示

| 关系 | 说明 | 符号 |
|------|------|------|
| 1:1 | 一对一 | 1 — 1 |
| 1:N | 一对多 | 1 — N |
| M:N | 多对多 | M — N |

## 6.3.3 时序图

### 时序图要素

| 要素 | 说明 |
|------|------|
| 对象 | 参与的类或角色 |
| 生命线 | 对象存在的时间 |
| 消息 | 对象间的交互 |
| 激活条 | 对象处理的时间 |

### 时序图用途
- 描述对象之间的交互顺序
- 表达业务流程
- 分析系统行为

## 6.3.4 状态机图

### 状态机图要素

| 要素 | 说明 |
|------|------|
| 状态 | 对象的某种状态 |
| 转换 | 状态之间的变化 |
| 事件 | 触发转换的事件 |
| 动作 | 转换时执行的动作 |

### 状态机图用途
- 描述对象的状态变化
- 分析对象生命周期
- 设计状态管理逻辑

## 数据建模与UML的关系

| 工作 | 说明 |
|------|------|
| 数据建模 | 关注数据实体和关系 |
| UML类图 | 表达类的结构 |
| ER图 | 表达数据库设计 |
| 时序图 | 表达对象交互 |

## 需求建模的重要性

**B端产品建模能力体现**
- 对业务本质的理解
- 对客观世界的抽象描述能力
- 决定了软件的灵活性和扩展性

**建模错误的代价**
- 后期调整成本高
- 可能导致系统重构
- 影响业务灵活性
