# B端产品化与SaaS战略

## 8.2 商业层面的产品化

### 产品化的三个层次

| 层次 | 说明 |
|------|------|
| 商业层面 | 标准化vs定制化的平衡 |
| 功能层面 | 核心功能的抽象与复用 |
| 技术层面 | 技术架构的灵活性 |

### SaaS产品化的挑战
- 标准化产品和个性化需求之间的平衡
- 不同客户的差异化诉求
- 如何通过配置满足大多数需求

## 8.3 功能层面的产品化与PaaS

### PaaS定义
- **P**latform as a **a** Service
- 甲方用户通过第三方提供的环境开发、维护、部署自己的软件系统
- 不用关心背后的基础设施

### PaaS的核心价值
- 提升产品的定制化能力
- 降低二次开发成本
- 支持多行业拓展

### 业务型产品的PaaS实践
- Oracle EBS、SAP
- Salesforce
- 北森、销售易

## 8.4 生态层面的产品化与Open API

### Open API的价值
- 实现系统对接
- 构建产品生态
- 扩展产品能力

### API设计原则
- RESTful API
- 接口文档完善
- 安全性保障

## 8.5 从定制化到产品化

### 定制化项目的问题
- 难以复制
- 开发成本高
- 维护困难

### 产品化路径
1. 抽象通用功能
2. 设计可配置方案
3. 积累行业模板
4. 构建PaaS平台

## 8.6 PaaS与低代码LCAP

### LCAP定义
- **L**ow **C**ode **A**pplication **P**latform
- Gartner于2019年提出
- 融合了低代码、aPaaS等概念

### 低代码/无代码对比

| 类型 | 说明 |
|------|------|
| Low Code | 通过拖拉拽等GUI交互界面与适量的微小代码开发应用系统 |
| No Code | 完全不用编码即可实现应用程序开发 |

### 核心观点
- SaaS产品必须聚焦，才可能实现标准化的产品设计
- 不论是业务垂直型还是行业垂直型，都是切入细分领域的方式
