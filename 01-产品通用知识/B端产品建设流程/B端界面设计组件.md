# B端产品界面设计组件

## 5.4 B端界面与常见组件设计

### 设计原则
- 交互模式固定
- 标准控件可以解决80%以上的业务场景
- 借鉴成熟软件设计方案

## 5.4.1 搜索组件设计

### 搜索功能类型

| 类型 | 说明 |
|------|------|
| 关键词搜索 | 针对单一字段的模糊搜索 |
| 高级搜索 | 多条件组合搜索 |
| 全局搜索 | 跨模块、跨数据对象的搜索 |

### 全局搜索设计要点
- 搜索范围覆盖所有模块
- 支持关键词高亮
- 提供搜索建议
- 搜索结果分类展示

### Salesforce全局搜索特点
- 字母命名的仪表盘和报表名称
- 支持多类型数据搜索
- 搜索结果智能分类

## 5.4.2 通知与消息设计

### 通知类型

| 类型 | 说明 | 特征 |
|------|------|------|
| 公告 | 内容比较重的通知形式 | 需要标题和富文本正文，用户点进去查阅 |
| 消息 | 轻量化的通知形式 | 只提醒事件发生，可嵌入链接跳转 |
| 待办 | 需要用户跟进处理的任务 | 有状态和过期时间 |

### 消息分类
- **系统消息**：由系统自动触发
- **人工消息**：由人工发送

### 设计要点
- 不同类型的消息应该通过唯一的消息组件统一管理
- 不应为不同类型消息创建不同形态的组件
- 待办和消息是完全不同的概念

## 5.4.3 待办组件设计

### 待办关键字段

| 字段 | 说明 |
|------|------|
| 描述 | 待办的具体内容 |
| 创建人 | 创建待办的人员 |
| 执行人 | 需要执行待办的人员 |
| 开始时间 | 控制待办显示的时机（可选） |
| 截止时间 | 逾期后可能触发业务逻辑 |
| 提醒时间 | 到期前多久进行提醒 |
| 待办状态 | 未开始、待处理、已超时、已完成 |

### 待办应用场景
- 销售跟进客户的提醒
- 运营人员交叉检查任务
- 可与系统中任意实体对象关联

## 5.4.4 日程组件设计

### 日程关键字段

| 字段 | 说明 |
|------|------|
| 主题 | 日程的名称 |
| 内容 | 描述具体日程事项 |
| 创建人 | 日程的创建者 |
| 参与者 | 会收到消息或邮件通知 |
| 开始/结束时间 | 精确到分钟 |
| 提醒时间 | 开始时间之前多久提醒 |
| 地点、附件 | 附加信息 |
| 重复规则 | 按周期重复设置 |

### 待办 vs 日程

| 维度 | 待办 | 日程 |
|------|------|------|
| 时间特性 | 有截止时间，处理完即消失 | 关注时间范围内的工作计划，始终存在 |
| 起始时间 | 一般没有 | 有 |
| 结束时间 | 有 | 有 |

## 5.4.5 列表页设计

### 列表页核心要素
- 检索条件区域
- 结果列表区域
- 分页器

### 列表页高级功能

| 功能 | 说明 |
|------|------|
| 自定义列 | 灵活配置显示字段 |
| 过滤器 | 保存常用查询条件 |
| 批量操作 | 批量编辑、删除、移动 |
| JQL查询 | 复杂查询语句支持 |

### JIRA列表页设计参考
- 支持任意查询条件配置
- 支持设置默认搜索值
- 支持自定义列表结果字段
- 支持JQL高级查询
- 支持批量操作

### 列表页设计常见Bug

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| 无全选按钮 | 批量操作需逐条勾选 | 添加全选按钮 |
| 搜索条件不匹配 | 有搜索项但列表无对应字段 | 确保显示字段与搜索条件匹配 |
| 分页技术风险 | 一次读取所有数据，数据库压力大 | 只加载当前页附近数据 |
| 数据泄露风险 | 可计算总数据量并导出 | 限制一次查询的数据量 |

### 分页设计建议
- 用户只能看到当前页前后几页数据
- 不能直接跳转到最后一页
- 不知道总共有多少页
- 在日期选择字段上进行控制

## 5.4.6 详情页设计

### 详情页组成
- 信息呈现区域
- 编辑轨迹
- 评论功能
- 附件追加

### 设计趋势
- 支持针对数据对象添加评论
- 支持@某人并发送提醒
- 类似Trello的任务卡片设计

## 5.4.7 界面设计建议

### 借鉴成熟软件
- Google Analytics
- 百度统计
- Salesforce
- 钉钉、飞书

### 使用标准控件
- Visio或Axure提供的控件即标准控件
- 不必创造标准控件以外的控件
- 节约产品经理和前端工程师时间

### 推荐学习资源
- **AntDesign**：蚂蚁金服UI标准，包含大量标准化控件设计
- **Salesforce LightningDesignSystem**：企业级软件体验设计思想
