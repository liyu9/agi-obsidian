# B端业务调研方法

## 3.5 B端业务调研的方法

B端业务调研的常用方法包括：
- 深度访谈
- 轮岗实习
- 调研问卷
- 数据分析
- 行业研究

### 3.5.1 深度访谈

深度访谈是了解业务全貌最快的手段。通过一对一面谈，可以直面问题，迅速获得答案。

**注意事项：**

**准备好访谈大纲**
- 提前准备好访谈思路、大纲、问题
- 选好访谈对象
- 想清楚通过访谈想要了解什么
- 如果事前没有做好准备，会导致对话内容发散、混乱

**从高级别人员开始访谈**
- 按照从概览到局部、从全局到细节的顺序研究业务
- 更容易把握整体调研工作的脉络和节奏
- 如果一开始就陷入细节的汪洋大海，会抓不住关键问题

**提前研究访谈对象**
- 访谈前要从各种渠道了解访谈对象的背景
- 针对高级别访谈对象，了解越充分越好
- 注意识别利益受损方，避免被干扰信息影响

**和访谈对象保持联系**
- 访谈结束后，最好和访谈对象建立长期联系
- 借助访谈的机会，拉近和业务人员的距离
- 后续项目中可以联系获取最真实的一线反馈

### 3.5.2 焦点小组

**定义**
- 挑选6～9人的用户作为一个小组
- 在组织人的控制下，不偏离主题地自由探讨
- 主持人观察人们的态度、反应

**应用场景**
- 常常用来做产品可用性测试(Usability Tests)
- 分析人员将新系统的交互设计方案进行现场演示
- 让小组人员针对旧方案和新方案分别进行可用性静默打分

### 3.5.3 轮岗实习

**定义**
- 产品经理深入一线，直接体验一线业务人员的具体工作
- 这是深入了解业务的最好方法

**核心要点**
- 最忌讲的就是凭自己的主观感觉进行设计，脱离实际
- 要么不断地和客户沟通、确认，要么直接和客户一起工作

**案例：催收系统优化**
- 老马在某金融企业做内部系统产品经理时
- 收到催收外呼系统改造需求后，没有马上开始写PRD
- 而是在催收部做了一周催收工作
- 发现80%的电话打不通才是效率低下的真正原因
- 解决方案：采用预测试外呼技术，系统自动呼出，客户接通后给业务人员弹屏提示
- 功能上线后，催收效率极大提升

**关键区别**
- 深入一线是产品经理有别于传统需求分析师的重要特征之一
- 产品经理要当一个冲在前线的人，而不是在后方拍脑袋的人

### 3.5.4 调研问卷

**注意事项：**

**激励用户完成填写**
- 控制问卷的长度，避免冗长乏味
- 在问卷开头告知设计目的、预计占用时间
- 最好能带一些活动礼物，提高参与积极性

**控制好开放式和封闭式问题**
- 建议大部分使用封闭式选择题，获取定量分析数据支撑
- 问卷结尾可以留1-2个开放式问题
- 如果开放式问题超过2个，需要谨慎思考每个开放式问题的意义

**避免诱导性问题**
- 不要问"我们要做一个更好的个人管理功能，您会使用吗？"
- 尽量采用中性描述，例如"您使用这款产品的体验如何？"
- 尽量不要设置非此即彼的问题

**主观感受数字化处理**
- 问法一：针对某功能X，您的感受是（非常喜欢/喜欢/无所谓/不喜欢/非常不喜欢）
- 问法二：针对某功能X，您的感受是（1-10打分）
- 第二种问法应用更广泛，通过计分方式将主观感受尽量客观化

**谨防"幸存者偏差"**
- 英军分析战斗结束后的飞机，发现机翼弹孔较多，判定要加强机翼装甲
- 实际上，战斗中坠落的飞机大都是因为腹部受攻击
- 幸存者偏差：幸存战斗机对分析工作产生了误导

### 3.5.5 数据走查

**关键指标分类**

| 指标类型 | 示例 |
|----------|------|
| 过程指标 | 新客户开户时长、订单处理时长、分拣配送时长、销售拜访量、线索转化率 |
| 结果指标 | 订单量、客单价、收入、成本、利润率 |

**核心要点**
- 产品经理要像业务经理那样关心业务运行的各项数据
- 这样才能了解业务现状，并进行业务诊断

## 3.6 制订一份调研计划

### 调研计划要点

| 要点 | 说明 |
|------|------|
| 合适的阶段 | 不同阶段采用不同的调研方法 |
| 合适的方法 | 根据调研目标选择调研手段 |
| 合适的用户 | 选择有代表性的访谈对象 |
| 正确的信息 | 确保获取的信息准确有效 |

### 调研计划表要素

| 要素 | 内容 |
|------|------|
| 调研目标 | 明确要了解什么 |
| 调研对象 | 谁是被调研的人 |
| 调研方法 | 访谈/问卷/轮岗等 |
| 调研时间 | 计划什么时候进行 |
| 预期输出 | 调研后要得到的结论 |

### M公司分销业务调研计划

**初期调研**
- 访谈公司高层（CEO、分销业务负责人）
- 了解大佬们对业务的展望、对系统的期待
- 果冻主要以学习为主

**中期调研**
- 分销业务运营、销售人员访谈
- 代表客户的管理、采购人员访谈
- M公司内部协作部门核心人员访谈
- 了解遇到的困难、挑战，对系统的期望

**调研目标**
- 通过和代表人员的访谈，了解业务大盘
- 整体运作情况、关键业务场景
- 典型业务问题和挑战
- 为产品下一步定位和规划工作打好基础
