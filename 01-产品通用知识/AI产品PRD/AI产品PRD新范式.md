# AI产品PRD新范式

## 传统PRD vs AI产品PRD

### 核心区别

| 维度 | 传统PRD | AI产品PRD |
|------|---------|-----------|
| 技术描述 | 功能逻辑、流程图 | 提示词设计、RAG架构 |
| 评测指标 | 转化率、留存率 | 准确率、召回率、F1 |
| 不确定性 | 较低 | 较高（AI效果不确定） |
| 合作模式 | 产品、研发、测试 | 产品、研发、算法 |

### AI产品PRD新增内容

1. **提示词设计文档**
   - 角色定义
   - 任务描述
   - 输出格式
   - 边界情况处理

2. **RAG架构设计**
   - 数据源
   - 分块策略
   - 检索方式
   - 重排序策略

3. **Agent设计**
   - 工具定义
   - 流程编排
   - 反思机制

4. **效果评估标准**
   - 测试数据集
   - 评测指标
   - 达标阈值

## 产研算法合作新范式

### 传统合作模式

```
产品 → 需求文档 → 研发 → 开发 → 测试 → 上线
```

### AI产品合作模式

```
产品 ←→ 算法：提示词调优
产品 → 研发：系统架构、数据处理
算法 → 研发：模型API对接
产品 ←→ 运营：效果监控、反馈收集
```

### 新分工

| 角色 | 职责 |
|------|------|
| 产品经理 | 提示词设计、效果评估、需求定义 |
| 算法工程师 | 模型选型、微调、效果优化 |
| 研发工程师 | 系统架构、API开发、数据处理 |

## 评测指标体系

### 分类任务指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 准确率 | TP/(TP+FP) | 预测正确的比例 |
| 召回率 | TP/(TP+FN) | 实际正确的被召回比例 |
| F1 | 2×P×R/(P+R) | 准确率和召回率的调和平均 |

### 生成任务指标

| 指标 | 说明 |
|------|------|
| BLEU | n-gram重叠度 |
| ROUGE | 召回导向 |
| Perplexity | 语言模型困惑度 |

### 业务指标

| 指标 | 说明 |
|------|------|
| 用户满意度 | 好评率、NPS |
| 问题解决率 | 能否解决用户问题 |
| 转化率 | 引导到目标行为的比例 |
