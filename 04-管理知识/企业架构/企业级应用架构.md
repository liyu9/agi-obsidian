# 企业级应用架构

## 13.2 学习企业级应用架构的益处

### 为什么B端PM需要了解企业架构
- 理解系统如何支撑业务运转
- 理解数据流转和依赖关系
- 做出更合理的产品设计决策

### 企业架构核心概念

| 概念 | 说明 |
|------|------|
| 业务架构 | 业务单元、组织结构、业务流程 |
| 应用架构 | 各软件系统之间的关系和边界 |
| 数据架构 | 数据存储、数据流转、主数据管理 |
| 技术架构 | 基础设施、技术选型 |

## 14.2 中型企业的应用架构

### 典型应用架构演进

| 阶段 | 系统 | 说明 |
|------|------|------|
| 初期 | 基础业务系统 | CRM、基础数据管理 |
| 成长 | 业务扩展 | 电商、客服、供应链 |
| 成熟 | 数据整合 | 数据仓库、主数据管理 |

### 主数据管理 (MDM)

**问题：信息孤岛**
- 多套客户资料库并存
- 数据不一致
- 无法形成完整客户视图

**MDM解决方案**
- 只保留一份客户信息库
- 各系统通过统一接口调用
- 核心客户属性统一管理
- 扩展属性由各系统维护

### 数据仓库 vs 大数据

| 维度 | 数据仓库 | 大数据 |
|------|----------|--------|
| 理念 | 交易数据，准确度高 | 日志行为数据，全量研究 |
| 技术 | 抽样、分析、预测 | 模型应用 |
| 用途 | 经营分析 | 业务自动化 |

## 15.1 集团企业的应用架构

### 多元化业务带来的挑战
- 多业务线、多公司类型
- 信息孤岛问题
- 基础功能重复建设

### 康威定律 (Conway's Law)
- 系统的架构是设计者组织结构的镜像
- 组织结构会影响系统架构的设计

### 电商业务架构示例

| 模块 | 说明 |
|------|------|
| 订单系统 | 订单管理 |
| 售后系统 | 退换货处理 |
| 客户信息 | 统一客户档案 |
| 会员系统 | 会员积分管理 |
| 营销系统 | 促销活动 |
| 账号系统 | 用户账户管理 |

### 主数据管理实施时机
- 引入MDM会让架构变复杂
- 实施初期需要较多时间和资源
- 需要在架构合理性和业务速度间权衡

## 15.2 加强基础服务建设

### 中台建设思路
- 将基础功能模块抽象出来
- 构建基础支撑体系
- 提高复用性，减少重复建设

### 中台类型

| 类型 | 说明 |
|------|------|
| 业务中台 | 订单、商品、会员等核心能力 |
| 数据中台 | 数据仓库、数据集市 |
| 技术中台 | 消息推送、权限管理 |
| 组织中台 | 共享服务中心 |

### 中台建设原则
- 业务导向
- 渐进式演进
- 服务复用
- 独立部署
