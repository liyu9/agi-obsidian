
## 一期：基础闭环期

### 本期核心目标

（1）完成现有已上线产品能力的 API 标准化，实现「无代码应用 - 数据表（管理数据表格）- 字段 - 记录」核心链路 API 闭环
（2） API 权限与员工现有产品权限天然对齐，无需额外权限配置
> 权限问题
一期不搭建独立可配置的 API 权限标识体系，所有 API 调用直接与企业员工现有产品权限强绑定，员工在产品内对无代码应用 / 数据表（管理数据表格）具备的操作权限，与 API 调用权限完全对齐，无产品操作权限则直接拦截 API 调用。

（3） 满足 dify、openclaw 集成的最低核心依赖，支撑基础 AI 数据读写、自动化执行场景
> 先提供API，再基于产品特性，封装对应的skill、MCP支持 AI 产品调用（成本低）

## 配套 dify、openclaw 分阶段对接策略

与 API 开发周期完全对齐，避免超前对接导致的返工：

1. 一期 API 上线后：完成 dify 基础集成（AI 数据读写、结构化数据生成）、openclaw 基础动作节点注册（手动触发型自动化）；
2. 二期 API 上线后：完成 dify 高阶场景集成（AI 视图 / 表单配置、附件内容解析）、openclaw 事件触发型自动化落地；
3. 三期 API 上线后：完成 dify+openclaw 深度整合，实现「自然语言指令→AI 生成配置→自动化流程执行」全链路闭环。
