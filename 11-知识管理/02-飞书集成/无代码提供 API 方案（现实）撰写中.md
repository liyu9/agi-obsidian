## 一期：基础闭环期

### 本期核心目标

（1）完成现有已上线产品能力的 API 标准化，实现「无代码应用 - 数据表（管理数据表格）- 字段 - 记录」核心链路 API 闭环
（2） API 权限与员工现有产品权限天然对齐，无需额外权限配置
> 权限问题
一期不搭建独立可配置的 API 权限标识体系，所有 API 调用直接与企业员工现有产品权限强绑定，员工在产品内对无代码应用 / 数据表（管理数据表格）具备的操作权限，与 API 调用权限完全对齐，无产品操作权限则直接拦截 API 调用。

（3） 满足 dify、openclaw 集成的最低核心依赖，支撑基础 AI 数据读写、自动化执行场景
> 先提供API，再基于产品特性，封装对应的skill、MCP支持 AI 产品调用（成本低）

### 一期 API 开发详细规划表

#### 1. 无代码应用管理

| API名称 | 作用 |
|--------|------|
| 创建无代码应用 API | 在指定文件夹中创建一个无代码应用，包含一个空白的数据表 |
| 获取无代码应用元数据 API | 获取指定无代码应用的元数据信息，包括无代码应用名称、版本号、是否开启高级权限等 |
| 更新无代码应用元数据 API | 更新无代码应用元数据，包括无代码应用的名称、是否开启高级权限 |
| 复制无代码应用 API | 复制一个无代码应用，可以指定复制到某个有权限的文件夹下 |

#### 2. 数据表（管理数据表格）管理

| API名称 | 作用 |
|--------|------|
| 新增一个数据表（管理数据表格）API | 新增一个数据表，支持传入数据表名称、视图名称和字段 |
| 新增多个数据表（管理数据表格）API | 新增多个数据表，仅可指定数据表名称 |
| 列出数据表（管理数据表格）API | 列出无代码应用中的所有数据表，包括其 ID、版本号和名称 |
| 更新数据表（管理数据表格）API | 更新指定数据表的信息 |
| 删除一个数据表（管理数据表格）API | 通过 app_token 和 table_id 删除指定的无代码应用数据表 |

#### 3. 字段管理

| API名称 | 作用 |
|--------|------|
| 新增字段 API | 在无代码应用数据表中新增一个字段 |
| 列出字段 API | 获取无代码应用数据表中的的所有字段 |
| 更新字段 API | 在无代码应用数据表中更新一个字段，更新时为全量更新 |
| 删除字段 API | 删除无代码应用数据表中的一个字段 |

#### 4. 记录管理

| API名称 | 作用 |
|--------|------|
| 新增记录 API | 在无代码应用数据表中新增一条记录 |
| 新增多条记录 API | 在无代码应用数据表中新增多条记录，单次调用最多新增 1,000 条记录 |
| 更新记录 API | 更新无代码应用数据表中的一条记录，为增量更新 |
| 删除记录 API | 删除无代码应用数据表中的一条记录 |

#### 5. 协作者基础管理

| API名称 | 作用 |
|--------|------|
| 新增协作者 API | 新增无代码应用高级权限中自定义角色的协作者 |

## 二期：扩展增强期

### 本期核心目标

1. 补全批量操作、条件查询等高频扩展 API，完善数据操作全场景能力
2. 完成权限体系全量 API 开发，实现自定义角色、协作者全生命周期管理
3. 匹配产品视图功能迭代计划，完成视图全量 API 开发（若开发视图）
4. 补全表单、事件回调等扩展能力，支撑 dify 高阶 AI 场景、openclaw 事件触发型自动化场景

| 列出记录 API | 获取数据表的记录列表 |
| 查询记录 API | 查询数据表中的现有记录，单次最多查询 500 行记录，支持分页获取 |
| 检索记录 API | 用于全文检索记录 |

#### 2. 协作者全量管理

| API名称 | 作用 |
|--------|------|
| 批量新增协作者 API | 批量新增无代码应用高级权限中自定义角色的协作者 |
| 列出协作者 API | 列出无代码应用高级权限中自定义角色的协作者 |
| 删除协作者 API | 删除无代码应用高级权限中自定义角色的协作者 |
| 批量删除协作者 API | 批量删除无代码应用高级权限中自定义角色的协作者 |

#### 3. 表单管理

| API名称 | 作用 |

| 列出表单问题 API | 列出表单中的所有问题项 |
| 更新表单问题 API | 更新表单中的问题项 |

#### 4. 字段编组管理

| API名称 | 作用 |
|--------|------|

#### 5. 数据表（管理数据表格）管理扩展

| API名称 | 作用 |
|--------|------|
| 删除多个数据表（管理数据表格）API | 通过 app_token 和 table_id 删除多个数据表 |

## 三期：高阶完善期

### 本期核心目标

1. 匹配产品流程 / 自动化功能迭代计划，完成流程引擎全量 API 开发
2. 补全剩余高阶 API，实现全量能力覆盖
3. 深度整合 dify、openclaw 能力，实现「无代码 + AI + 自动化」全链路闭环
4. 完善企业级合规审计、高可用保障体系，支撑大规模内部场景落地

| API 调用审计日志查询 API | 用于查询 API 调用审计日志 |

#### 4. 视图管理

| API名称 | 作用 |
|--------|------|
| 新增视图 API | 在无代码应用数据表中新增一个视图，可指定视图类型 |

#### 5. 自定义角色管理

## 配套 dify、openclaw 分阶段对接策略

与 API 开发周期完全对齐，避免超前对接导致的返工：

1. 一期 API 上线后：完成 dify 基础集成（AI 数据读写、结构化数据生成）、openclaw 基础动作节点注册（手动触发型自动化）；
2. 二期 API 上线后：完成 dify 高阶场景集成（AI 视图 / 表单配置、附件内容解析）、openclaw 事件触发型自动化落地；
3. 三期 API 上线后：完成 dify+openclaw 深度整合，实现「自然语言指令→AI 生成配置→自动化流程执行」全链路闭环。
