## 多维表格官方版API
[[多维表格API列表]]
多维表格（Base）是全新的业务管理工具，帮助用户重构工作应用和团队协同模式，高效在线协同数据，随心构建个性化应用，轻松掌控全盘业务数据，和团队一起创造效率的无限可能。多维表格可以是一个表格，也可以是无数个应用。它拥有强大的底层开放能力，你可以通过多维表格 API 轻松打通内部其他业务系统，让业务数据通畅流转，实时同步。

# API 使用限制
一、接口操作核心限制

1. **批量操作限制**：针对记录的批量操作，单次请求最多可处理**1000 条记录**；批量操作的执行结果为**完全成功或完全失败**，无部分执行成功的情况。
2. **写入操作稳定性约束**：为保障 Base 服务稳定性，官方建议**单个 Base 同一时间仅执行 1 次 API 写入操作**，避免并发写入引发服务异常。

单个 Base 内资源数量上限

官方明确的单个 Base 内各资源的最大限制如下表：

表格

| 资源类型                | 最大限制                               | 补充说明                                           |
| :------------------ | :--------------------------------- | :--------------------------------------------- |
| 记录（Records）         | 随租户权限变化，平台无额外强制限制，以 Base UI 界面提示为准 | 常规在线使用上限为 50000 条记录；开启归档表功能后，最多可归档 1000000 条记录 |
| 字段（Fields）          | 300 个                              | 其中公式字段最多支持 100 个                               |
| 视图（Views）           | 200 个                              | -                                              |
| 数据表（Tables）         | 100 个                              | -                                              |
| 自定义角色（Custom Roles） | 30 个                               | -                                              |
| 高级权限协作者             | 200 个                              | -                                              |
